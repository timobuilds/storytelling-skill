# Chapter 6: Function Tactics

## Core Idea
Function tactics answer *what the story needs to do*: sell, pitch, surface insights in conversation, or warm up a room.

## Frameworks Introduced

- **Pitch Perfect** (Desert Island): Convince someone to back you — problem, solution, trust.
  1. One-sentence top line (use What's It About?).
  2. **Elevator**: Problem for me? How do you solve it? Why trust you? (Trust Me, I'm an Expert).
  3. **POPP arc** for drama:
     - **P**roblem (negative) — what needs fixing now
     - **O**pportunity (positive) — also available now
     - **P**ractical Steps (negative) — won't be easy
     - **P**romise (positive) — if we get it right, great
  4. Visual pitch via Show & Tell.
  - Prepare with Audience Profile.

- **Simple Sales Stories**: "Someone like you buys this" or "Someone you like buys this."
  - Also: Old / Regular / Tricky customer stories (longevity, repeat, recovery).
  - Pair with Audience Profile + Social Proof; develop with No Easy Way or Pitch Perfect.

- **Story-ish Conversations**: Make everyday talk yield stories.
  1. Vivid/visual moments → Movie Time
  2. Who/what/where/when → Story Listening
  3. Emotion → Emotional Dashboard
  4. Conflict → Three Great Conflicts
  5. Change & new info → Five Ts

- **Icebreaker Stories**: Warm up story muscles.
  - **Photo Story**: 3 images → Before / During / After.
  - **Love and Hate**: Same object, opposite feelings → invent why (builds drama + emotion).

## Key Concepts
- **POPP**: Problem → Opportunity → Practical Steps → Promise.
- **Someone like you / someone you like**: Two social-proof sales story morals.
- **Story-ish**: Bias conversations toward people, change, emotion, conflict.

## Mental Models
- Use Pitch Perfect for investment, boss buy-in, or any "tenth pitch of the day."
- Use Simple Sales Stories when prospects need peer evidence.
- Use Story-ish Conversations when meetings are stuck in abstraction.
- Use Icebreakers before workshops that need visual/dramatic thinking.

## Anti-patterns
- **Pitching in a literal elevator** (weird — and conceptually, rushing without POPP).
- **Feature dump** instead of problem → trust → promise.
- **Sales stories about dissimilar customers** without bridging sector/problem.

## Key Takeaways
1. Every pitch needs problem, solution, and a trust story.
2. POPP creates memorable rise/fall better than flat benefit lists.
3. Sales stories work via similarity or admired influencers.
4. You don't need slides to practice Function — conversation is enough.
5. Desert Island Function card: **Pitch Perfect**.
