# Storyteller Start

## When this skill owns the turn

The user’s ask is broad, multi-stage, or vague (“help me tell this story”). Your job: diagnose → **propose one recipe skill by name** → start with a filled draft artifact.

## Doctrine

Read [../storyteller-tactics/references/doctrine.md](../storyteller-tactics/references/doctrine.md). Enforce: **diagnose the job → one recipe → draft a story they can see.**

## Draft-first

Propose the recipe’s **step-1 artifact** filled from context (use that skill’s `assets/` template; mark `TBD`). Match the density of [assets/exemplar-story-intake.md](assets/exemplar-story-intake.md) and the target recipe’s exemplar. Ask the user to react — **don’t** open with a blank questionnaire or the seven “Do you know…?” cards as an interview script.

Routing table: [references/recipes.md](references/recipes.md).

## Recipe first (vague asks)

If the ask is vague, pick **one** recipe from [references/recipes.md](references/recipes.md) and say why. Then begin that recipe’s process with a filled draft.

| User signal | Skill |
| --- | --- |
| Sell / convert / case study for buyers | `stories-that-sell` |
| Buy-in / change / leap of faith | `stories-that-motivate` |
| Expertise / data / why trust my judgment | `stories-that-convince` |
| Empathy / research / know the human | `stories-that-connect` |
| Strategy / why this direction | `stories-that-explain` |
| Team culture / how we work | `stories-that-lead` |
| Deck / keynote / presentation | `stories-that-impress` |

## Diagnose (only what’s missing)

1. **Job:** sell, motivate, convince, connect, explain, lead, impress?
2. **Audience:** who hears this? hopes / fears known?
3. **Surface:** pitch, blog, all-hands, sales call, deck, 1:1?
4. **Proof:** whose story is it? permission? customer evidence?
5. **Risk:** does “new” scare them? (Innovation Curve)

## Single-skill route (clear asks)

If they already name the job or a tactic family, load that recipe skill immediately and skip re-routing theater. If they name a single tactic (e.g. Man in a Hole), use `storyteller-tactics` and apply it.

## Output

1. **Diagnosis** (2–4 bullets)
2. **Primary skill** + why (one recipe)
3. **Hard no** (what you won’t do this turn — e.g. “won’t write slide polish before Audience Profile”)
4. **Start now:** filled draft of step-1 artifact (`TBD` for unknowns)
5. Tell the user you’ll continue under that recipe skill’s process

## Anti-patterns

- Asking 10 clarifying questions before proposing a recipe
- Running all seven recipes “just in case”
- Jumping to Show & Tell / slide craft before job + audience
- Treating `storyteller-tactics` as the front door for vague asks
